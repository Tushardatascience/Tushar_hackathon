# Tushar_hackathon
# Project Schedule Designer – ML + LLM Solution

## 1. Project Overview

The goal of the system is to automatically generate a **recommended project schedule** for a new construction project using historical project data.

The system combines:

* **Machine Learning similarity detection**
* **Schedule retrieval from historical data**
* **Large Language Model (LLM) schedule generation**

The final pipeline generates a structured project schedule based on similar historical projects.

---

# 2. Problem Statement

Construction scheduling requires significant domain knowledge and experience with similar past projects.

This solution addresses the problem by:

1. Accepting **new project metadata**
2. Identifying **similar historical projects**
3. Retrieving their **existing schedules**
4. Using an **LLM to generate a recommended schedule**

---

# 3. System Architecture

End-to-end pipeline:

New Project Input
→ Feature Engineering
→ Similarity Detection (ML Model)
→ Retrieve Similar Historical Projects
→ Extract Their Schedules
→ LLM Prompt Construction
→ Recommended Project Schedule

---

# 4. Dataset Description

Two datasets are used in this project.

### job_meta_data.csv

Contains project-level metadata.

### schedule_data.csv

Contains task-level schedule data for projects.

# 5. Methodology

## Step 1 – Data Cleaning

* Handled missing values
* Standardized categorical variables
* Removed duplicate records
* Ensured consistency between project IDs

## Step 2 – Feature Engineering

Categorical variables were converted into numerical format using **Label Encoding**.

Important features used:

* Region
* Market Sector
* Business Unit
* Project Type
* Project Size / Cost

## Step 3 – Feature Scaling

Numeric features were scaled using **StandardScaler** to ensure fair similarity comparisons.

## Step 4 – Similarity Detection

A **Cosine Similarity model** was used to identify projects most similar to the new project.

Top N similar projects are selected.

## Step 5 – Schedule Retrieval

Schedules for similar projects are retrieved from `schedule_data.csv`.

These schedules provide historical context.

## Step 6 – LLM Integration
The model used:

Azure OpenAI – GPT-4o
